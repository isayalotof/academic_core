# Запуск фронтенда для разработки

## Быстрый старт

1. **Установите зависимости** (если еще не установлены):
```bash
cd frontend
npm install
```

2. **Создайте файл `.env`** (если его еще нет):
```bash
# Скопируйте пример
cp env.example .env
```

Или создайте файл `.env` вручную со следующим содержимым:
```
VITE_API_URL=http://localhost:8000
VITE_MAX_BOT_TOKEN=your_bot_token_here
```

3. **Запустите dev-сервер**:
```bash
npm run dev
```

Приложение будет доступно по адресу: **http://localhost:3000**

## Настройки

### Порт
По умолчанию используется порт `3000`. Если он занят, Vite автоматически предложит использовать следующий свободный порт.

Чтобы изменить порт вручную, отредактируйте `vite.config.ts`:
```typescript
server: {
  port: 3001, // ваш порт
}
```

### API URL
URL бэкенда настраивается через переменную окружения `VITE_API_URL` в файле `.env`.

По умолчанию: `http://localhost:8000`

### Hot Reload
Vite автоматически перезагружает страницу при изменении файлов. Изменения применяются мгновенно без полной перезагрузки страницы.

## Полезные команды

- `npm run dev` - запуск dev-сервера
- `npm run build` - сборка для продакшена
- `npm run preview` - предпросмотр собранного приложения
- `npm run lint` - проверка кода линтером

## Решение проблем

### Порт занят
Если порт 3000 занят, Vite автоматически предложит использовать другой порт. Или измените порт в `vite.config.ts`.

### Ошибки при установке зависимостей
```bash
# Очистите кэш и переустановите
rm -rf node_modules package-lock.json
npm install
```

### API не доступен
Убедитесь, что:
1. Бэкенд запущен на `http://localhost:8000`
2. В файле `.env` указан правильный `VITE_API_URL`
3. Нет проблем с CORS на бэкенде
